Kârlılık Mantığı

Para değerleri her zaman integer cent/kuruş olarak saklanır. Float kullanılmaz.

  • Gelir = ödenmiş fatura toplamı
  • Saat maliyeti = toplam saat × kullanıcının saatlik maliyeti
  • Net kâr = gelir - saat maliyeti - giderler
  • Kâr marjı = net kâr / gelir × 100

Risk Skoru

0-30 sağlıklı, 31-60 dikkatli, 61-100 zararlı müşteri. Skor; gecikmiş fatura, ödeme gecikmesi, şikayet/revizyon oranı, kâr marjı ve iş hacmiyle hesaplanır.

Event, Cache ve Monitoring

Bu bölüm canlı sistem davranışını hızlı anlamak için hazırlanmıştır.

Kafka Eventleri

  • time.entry.created → profitability recalculation
  • expense.added → profitability recalculation
  • invoice.paid → profit + risk update
  • invoice.overdue → risk increase
  • project.closed → final snapshot

Redis Cache Keys

  • profit:project:{project_id}
  • profit:client:{client_id}
  • risk:client:{client_id}
  • dashboard:user:{user_id}

Observability

  • /metrics tüm servislerde açık
  • http_requests_total
  • http_request_duration_seconds
  • kafka_events_published_total
  • kafka_events_consumed_total

Happy Path

  • Register + login

Security Path

  • Token olmadan /dashboard çağır → 401
  • Yanlış token ile /clients çağır → 401
  • Başka user verisine erişim dene → 404/403 bekle
  • Refresh token ile access yenile
